smallertalk

This is a yet-to-be-implemented demake of the programming language Smalltalk, which is heavily based on the object-oriented paradigm. In fact, it's so object-oriented, it almost feels a bit... functional. Of course, state is basically 1/3rd of the language:

A Smalltalk object can do exactly three things:
  1. Hold state (references to other objects).
  2. Receive a message from itself or another object.
  3. In the course of processing a message, send messages to itself or another object.
Although, the other 2/3 seems to be functional, even though that's an imperative thing. ok, i'll stop talking about that.

First, let's talk about the main thing: objects. I'm gonna take an approach similar to Squeak (the most mainstream Smalltalk dialect!), and make a class a directory. The directory will contain other inherited classes as directories, and methods as files. The names of these will be the names of the classes and methods. Here's an example, where - is a level, and $ is a file instead of a directory: 

	Object
	- Magnitude
	-- Number
	--$ +;x;
	--$ -;x;
	--- Float
	---$ round;
	--- Integer
	---$ isDivisibleBy:
	---$ toBase:
As you can see, the : suffix behaves like Squeak, while the ; suffix is invisible. So, it's 3 toBase: 2, but it's 3 + 4. I guess this also means composition is just symbolic links... You can escape colons, but you have to do \:, which for you is \\:. You can prefix file names with ^ to make them private to the object from it's subclass object(s). Calling a class by itself (e.g. to set a variable to) uses the init or init: method.

Arbritrary amounts of input should be manhandled (manually handled). But you know what else should be manhandled? returning. Instead of self and thisContext, we have this. We can call this :: 1 to become, and this := x 1 to hold, reminiscent of functio—Oh, wow, haha. Becoming doesn't change the object's class, just that object. The Object class will have a hasHeld: for variables, that returns false by default, and has to be changed by inheritors. (Don't worry, true and false aren't psuedovariables, they're just normal variables in the Object class.) Also, super can be avoided by An OrderedCollection, and this actually refers to the local space. So, 

[:a :b | this :: a + b;]
works like ^. Actually, you know what? I'll make blocks take exactly one argument, and becuase of that you don't need the : prefix or the |. So, it's now 
[a [b this :: a + b;]]

Let me just list some more things. Strings are just arrays of characters, so you don't need them. ; replaces . as the command separator. && replaces ; as the method separator.
